GateGuard → Entry & Exit Sequence Diagram

IIT Gandhinagar

Design Principle
The Guard is the decision-maker. The Guard manually verifies the person's ID card (Student ID / Aadhaar / DL), approves entry, then logs it into the system. The system is a logger, not a gatekeeper. Unknown vehicle plates are auto-registered. VehicleID = NULL means the person came on foot.
1 Person Entry with Vehicle
Flow Summary
Guard verifies ID card visually → approves → enters memberID + license plate + vehicle type (dropdown) → system finds or creates vehicle → logs PersonVisit + VehicleVisit + updates occupancy.

3 Person Exit
Flow Summary
Guard enters memberID → system finds active visit → updates PersonVisit exit → if vehicle was linked, updates VehicleVisit exit too → decrements occupancy.

Key Design Decisions
  • Guard decides entry (visual ID check) → system only logs
  • Unknown vehicle plate → auto-register (INSERT new Vehicle)
  • VehicleID = NULL → person came on foot
  • Guard selects vehicle type from dropdown (PrivateCar, Bike, Taxi, Truck, Bus)
  • Entry and exit gates can be different (separate FK columns)
